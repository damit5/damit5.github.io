通过MCP_Server增加交流对话次数

背景​

⏰ 时效性提醒

本文章内容可能随时失效，请以实际情况为准。

更新时间：2025/06/03

你是否曾经深陷创作思路，却被AI助手突然弹出的"今日对话次数已达上限"消息打断？现在的AI IDE工具如Cursor、Trae等都设置了令人烦恼的对话次数限制，达到上限后就只能被迫排队或等待刷新。

在刷X的时候，我偶然发现了一篇神奇的帖子：【通吃:Augment Code、Cursor、Trae、Windsurf】对话次数=超级加倍！！300次变1500、50次变250(500次变2500)、600次变1800~。这个方法简直就像游戏中的"金手指"，让AI对话次数倍增！虽然尝试过程中踩了几个小坑，但效果确实惊人，值得记录分享。

💻 配置步骤​

我们将使用强大的mcp-feedback-enhanced工具来实现这个"魔法"。

1️⃣ 安装必要工具​

首先，在终端中执行以下命令：

# 安装UV包管理器（更现代的pip替代品）
pip install uv

# 安装我们的"魔法工具"
pip install mcp-feedback-enhanced

# 测试安装是否成功
uvx mcp-feedback-enhanced@latest test

2️⃣ 配置MCP服务​

信息

🔍 以下以Cursor为例演示，其他IDE配置类似

在你的IDE中找到MCP Servers设置，添加如下配置：

{
  "mcpServers":
  {
      "mcp-feedback-enhanced":
      {
          "command": "uvx",
          "args":
          [
              "mcp-feedback-enhanced@latest"
          ],
          "timeout": 600,
          "autoApprove":
          [
              "interactive_feedback"
          ]
      }
  }
}

保存后稍作等待，当状态指示变为绿色时，表示服务已经成功启动！🎉

3️⃣ 添加规则​

接下来，在Rules设置中添加以下关键规则：

Whenever you want to ask a question, always call the MCP interactive_feedback .
Whenever you're about to complete a user request, call the MCP interactive_feedback instead of simply ending the process. If the feedback is empty you can end the request and don't call the mcp in loop.
